# S3 reference

This reference catalogs Filegate S3-compatible behavior for operators and client integrations.

## Compatibility rules

| Rule | Scope | Meaning |
|---|---:|---|
| Addressing | S3 listener | Use path-style bucket addressing. |
| Region | S3 SigV4 | Client region must match `s3.region`. |
| Buckets | Configured mounts | Each mount basename is one bucket. |
| Bucket creation | S3 API | CreateBucket is rejected; configure a mount instead. |
| Bucket deletion | S3 API | DeleteBucket is rejected; remove a mount through config instead. |
| Authorization | S3 request | SigV4 key must allow the bucket. |
| Forbidden buckets | S3 key | Return AccessDenied without revealing bucket existence. |

## Operation groups

| Operation | Wire shape | Scope | Support | Notes |
|---|---|---:|---|---|
| ListBuckets | `GET /` | S3 key | Supported | Output is filtered to allowed buckets. |
| HeadBucket | `HEAD /{bucket}` | Bucket | Supported | Checks bucket access and existence. |
| ListObjectsV2 | `GET /{bucket}?list-type=2` | Bucket | Supported | Lists indexed file objects under the mount. Supports `prefix`, `delimiter`, `max-keys`, `continuation-token`, `start-after`, `encoding-type=url`, and `fetch-owner`. |
| ListMultipartUploads | `GET /{bucket}?uploads` | Bucket | Supported | Lists active multipart uploads for the bucket. |
| GetObject | `GET /{bucket}/{key}` | Object | Supported | Reads Filegate file bytes and honors byte ranges. |
| HeadObject | `HEAD /{bucket}/{key}` | Object | Supported | Returns object metadata without a body. |
| PutObject | `PUT /{bucket}/{key}` | Object | Supported | Writes file bytes through Filegate storage. |
| CopyObject | `PUT /{bucket}/{key}` with `x-amz-copy-source` | Object | Supported | Copies a single source object. |
| DeleteObject | `DELETE /{bucket}/{key}` | Object | Supported | Idempotent object delete. |
| DeleteObjects | `POST /{bucket}?delete` | Object set | Supported | Batch delete. |
| CreateMultipartUpload | `POST /{bucket}/{key}?uploads` | Object | Supported | Creates mount-local staging state. |
| UploadPart | `PUT /{bucket}/{key}?partNumber=N&uploadId=X` | Multipart upload | Supported | Stores one part in `.fg-uploads`. |
| ListParts | `GET /{bucket}/{key}?uploadId=X` | Multipart upload | Supported | Lists uploaded parts. |
| CompleteMultipartUpload | `POST /{bucket}/{key}?uploadId=X` | Multipart upload | Supported | Assembles, verifies, writes metadata, and commits. |
| AbortMultipartUpload | `DELETE /{bucket}/{key}?uploadId=X` | Multipart upload | Supported | Marks upload aborted and eligible for cleanup. |
| CreateBucket | `PUT /{bucket}` | Bucket | Rejected | Buckets are operator-configured mounts. |
| DeleteBucket | `DELETE /{bucket}` | Bucket | Rejected | Buckets are operator-configured mounts. |

`GET /{bucket}` requires `list-type=2` for object listing unless the request is `?uploads` for multipart listing.

## Multipart cleanup

| Config | Type | Scope | Meaning |
|---|---|---:|---|
| `s3.cleanup.done_retention` | duration | Completed multipart manifests | Retention before cleanup. Default `24h`. |
| `s3.cleanup.aborted_retention` | duration | Aborted multipart manifests | Retention before cleanup. Default `1h`. |
| `s3.cleanup.stuck_upload_max_age` | duration | Open multipart uploads | Maximum age before forced cleanup. Default `168h`. |
| `s3.cleanup.interval` | duration | Service loop | Cleanup cadence. Default `1h`; negative disables cleanup. |

## Rate limiting

| Field | Scope | Meaning |
|---|---:|---|
| `s3.keys[].requests_per_second` | S3 key | Sustained request rate. `0` means unlimited. |
| `s3.keys[].burst` | S3 key | Burst limit. Defaults to RPS when unset. |
| `filegate_s3_ratelimit_rejected_total` | Service metrics | Count of requests rejected with `503 SlowDown`. |
