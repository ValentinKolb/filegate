# S3 compatibility

The S3-compatible listener is for tools that can speak path-style S3 but should store bytes on Filegate mounts.

## Enable S3

```sh
sudo fg config set --config /etc/filegate/conf.yaml \
  --s3-enabled \
  --s3-listen ':9100' \
  --s3-region us-east-1

sudo fg config s3 key add --config /etc/filegate/conf.yaml \
  --all-buckets

sudo systemctl restart filegate
```

The key command prints the generated access key and secret key once. Store them as secrets.

## Bucket model

| S3 concept | Filegate concept | Scope |
|---|---|---:|
| Bucket | Configured storage mount basename | Per mount |
| Object key | File path inside a mount | Per object |
| Multipart upload | Mount-local staging under `.fg-uploads` | Per upload |
| Access key | S3 credential entry in config | Per key |
| Bucket allowlist | `s3.keys[].buckets` | Per key |

CreateBucket and DeleteBucket are rejected. Operators create buckets by configuring mounts.

## Client example

```sh
aws --endpoint-url http://127.0.0.1:9100 s3 ls s3://data/
```

Use path-style addressing. Configure the client endpoint to the S3 listener, not the REST listener.

## Multi-tenant keys

| Field | Scope | Meaning |
|---|---:|---|
| `access_key` | S3 key | SigV4 access key ID. |
| `secret_key` | S3 key | SigV4 secret. |
| `buckets` | S3 key | Allowed bucket names. `*` grants every configured mount. |
| `requests_per_second` | S3 key | Sustained request rate limit. `0` means unlimited. |
| `burst` | S3 key | Burst size. Defaults to RPS when unset. |

Requests to buckets outside the key allowlist return AccessDenied without revealing bucket existence.

## Supported operation groups

| Group | Scope | Notes |
|---|---:|---|
| Bucket listing | S3 listener | ListBuckets is filtered by key allowlist. |
| Object read/write/delete | Bucket object | Uses Filegate filesystem storage and metadata index. |
| Multipart upload | Bucket object | Parts are staged under `.fg-uploads` and completed atomically. |
| CopyObject | Bucket object | Uses Filegate copy behavior. |
| DeleteObjects | Bucket object set | Batch delete support. |

See [S3 reference](reference/s3) for supported operations and deviations.
